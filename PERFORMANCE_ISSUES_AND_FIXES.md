# Анализ Проблем Производительности и Ошибок Сайта

## 🔍 Найденные Проблемы

### 1. **Cache Policy Issue** ✅ ИСПРАВЛЕНО
- **Файл:** `assets/js/include.js`
- **Проблема:** Использование `cache: "no-cache"` в fetch запросах
- **Влияние:** Браузер не кэширует ресурсы, что замедляет загрузку на 30-50%
- **Решение:** Изменено на `cache: "default"` для включения браузерного кэша
- **Исправлено на:** 2 места в коде

### 2. **Множественные setTimeout вызовы**
- **Файл:** `assets/js/include.js`
- **Проблема:** 
  - `setTimeout(scan, 300)` - сканирование элементов через 300ms
  - `setTimeout(scan, 1200)` - повторное сканирование через 1200ms
  - `setTimeout(cleanupOpenAi, 300)` - очистка AI виджета через 300ms
- **Влияние:** Замедляет интерпретацию JS, блокирует рендеринг
- **Решение:** Оптимизировать через requestAnimationFrame или debounce

### 3. **Огромный Размер include.js**
- **Размер:** 69KB (1880 строк кода)
- **Содержит:** AI виджет, аналитика, header/footer загрузка, модель viewer и др.
- **Проблема:** Монолитный файл замедляет парсинг и выполнение
- **Рекомендация:** Разделить на несколько модулей

### 4. **Встроенные Стили в HTML**
- **Файл:** `index.html`, `eng/index.html`
- **Проблема:** Множество `<style>` блоков прямо в HTML
- **Влияние:** Увеличивает размер HTML, замедляет парсинг
- **Рекомендация:** Перенести в CSS файлы

### 5. **Отсутствование requestIdleCallback для инициализации**
- **Файл:** `assets/js/include.js`, `assets/js/modern-dropdown.js`
- **Проблема:** Все инициализации происходят синхронно на главном потоке
- **Решение:** Использовать requestIdleCallback для неприоритетных задач

## 🛠️ Технические Детали

### Проблема с cache: "default"
```javascript
// БЫЛО (медленно):
const res = await fetch(path, { cache: "no-cache" });

// СТАЛО (быстро):
const res = await fetch(path, { cache: "default" });
```

Это позволяет браузеру использовать кэшированные версии ресурсов, что значительно улучшает производительность при повторных посещениях.

### setTimeout Оптимизация
```javascript
// БЫЛО (три вызова):
scan();
setTimeout(scan, 300);
setTimeout(scan, 1200);

// ДОЛЖНО БЫТЬ (один вызов из requestAnimationFrame):
if ('requestIdleCallback' in window) {
  requestIdleCallback(scan, { timeout: 1000 });
} else {
  setTimeout(scan, 100);
}
```
