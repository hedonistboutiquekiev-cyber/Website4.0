## 📋 Что было исправлено

### Основная проблема
Выпадающие меню (УСЛУГИ, ALBAMAN, О НАС) отображались позади контента и были невидимы пользователям.

### Решение
Были внесены комплексные изменения в CSS для обеспечения корректного отображения меню:

1. **Добавлено `overflow: visible`** на все контейнеры header:
   - .site-header
   - .header-inner
   - .main-nav
   - .dropdown

2. **Установлена правильная Z-Index иерархия:**
   - .dropdown-menu: z-index: 2147483647 (максимум)
   - .site-header: z-index: 2147483647
   - .header-inner: z-index: 2147483647
   - .main-nav: z-index: 2147483646
   - .dropdown: z-index: 2147483646

3. **Оптимизирована высота header:**
   - Изменена с фиксированной 80px на auto (desktop)
   - Добавлено min-height: 80px для визуальной консистентности

4. **Улучшена видимость меню:**
   - Добавлены opacity переходы
   - Оптимизирована работа visibility и pointer-events
   - Добавлены плавные transition эффекты

## 💡 Почему это работает

### Z-Index иерархия
```
2147483647 ← DROPDOWN-MENU, SITE-HEADER, HEADER-INNER
2147483646 ← MAIN-NAV, DROPDOWN
   1000    ← Page modals
    100    ← Page content
      0    ← Background
```

### Overflow свойства
- `overflow: visible` на родительских контейнерах позволяет дочерним элементам выходить за границы
- Это критично для выпадающих меню, которые позиционируются относительно родителя

### Позиционирование
- `position: relative` на контейнерах создаёт стекинг контекст
- Это требуется для правильной работы z-index у дочерних элементов
